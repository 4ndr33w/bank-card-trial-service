# Система управления банковскими картами
<hr/>

## Описание:
Тестовое задание по созданию системы управления банковскими картами. <br>
Система предоставляет REST API для администрирования и менеджмента пользователей <br>
и операций с банковскими картами пользователей.
<hr/>

## Функционал:
<hr/>

### 1) Неавторизованные пользователи:
   * имеют возможность зарегистрироваться в системе.

### 2) Пользователи с ролью `USER` имеют возможность:
   * просматривать свои карты: просматривать данные конкретной карты или список постранично.
   * Выполнять запрос на блокировку карты.
   * переводить средства между своими картами.
   * просматривать баланс карты.
   * обновлять, удалять и блокировать и просматривать свой профиль.

### 3) Пользователи с ролью `ADMIN` имеют возможность:
   * создавать, обновлять, удалять, блокировать и разблокировать карты пользователей.
   * просматривать список карт пользователей - как карты отдельного клиента, так и все карты постранично.
   * обновлять, удалять, блокировать, разблокировать профиль пользователя.
   * Добавлять и удалять роли пользователя.

### Запуск:

Для запуска сервиса необходимо выполнить выполнить следующие шаги:<br>
(Для локального запуска первые 3 шага можно пропустить - в коммит включены уже готовые ключи, <br>
но в продакшене нужно будет сгенерировать новые ключи. <br>
Также в коммит включен дефолтный `.env`-файл с дефолтной конфигурацией для локального запуска)

### 1) Установить `OpenSSL`
например:
```bash
winget install ShiningLight.OpenSSL.Light
```

### 2) Сгенерировать ключи для подписи токенов:
Можно использовать уже имеющиеся, но для безопасности лучше сгенерировать новые. <br>
Необходимо зайти в каталог `src/main/resources/keys` и выполнить в нём команды:

Для Access Token:
```bash
openssl genrsa -out access_private.pem 2048
```
```bash
openssl rsa -in access_private.pem -pubout -out access_public.pem
```
Для Refresh Token:
```bash
openssl genrsa -out refresh_private.pem 2048
```
```bash
openssl rsa -in refresh_private.pem -pubout -out refresh_public.pem
```

### 3) Удалить префиксы и постфиксы из ключей:
Из каждого ключа удалить префиксы и постфиксы (-----BEGIN PRIVATE KEY----- / -----END PRIVATE KEY-----) <br>
Как для приватных, так и для публичных ключей

### 4) Установить и/или запустить Docker:
```bash
winget install Docker.DockerDesktop
```

### 5) Запустить в докере базу данных:
В каталоге проекта выполнить команду:
```bash
docker-compose up bank-db
```
### 6) Выполнить компиляцию и сборку файлов проекта:
```bash
mvn clean install
```

### 7) Запустить сервис в Docker:
```bash
docker-compose up bank-rest-service
```

### Адрес API:
`url`: http://localhost:700 <br>
`Swagger`: http://localhost:700/swagger-ui/index.html`
<hr/>

### Swagger:
Поддерживает 2 типа авторизации: `Basic` и `Bearer` <br>
#### Basic:
При вводе логина и пароля, будет доступен `endpoint` для получения `JWT` токена. <br>
адрес ресурса: http://localhost:700/api/v1/login <br>
(через Postman выбираем `Basic Auth`)
#### Bearer:
Остальные ресурсы (кроме ресурса создания пользователя) требуют `JWT` токена.

#### Ресурс создания пользователя:
`url`: http://localhost:700/api/v1/users метод `POST` <br>
<hr/>
